# Session Recap — 2026-04-05

**Project:** academic-works-harvest
**Session focus:** Harvest CUNY Academic Works OAI-PMH records in both oai_dc and qdc, analyze where document type lives, and decide how to split harvesting between formats for Primo VE (OneSearch) ingest.

## What we built / changed
- Built `src/harvest.py` — OAI-PMH harvester that pulls all 269 sets in both `oai_dc` and `qdc` metadata formats, follows resumption tokens, skips already-downloaded files. Harvested ~133K records in each format.
- Built `src/analyze_types.py` — scans both format directories, reports field presence by file, aggregates value frequencies, compares against the 42 canonical Academic Works document types, generates a 4-sheet Excel report (`data/type_field_analysis.xlsx`).
- Re-harvested `publication:ny_pubs` and `publication:pubs` in qdc after initial timeouts/502 errors — turned out to be complete anyway (errors hit on the post-last-page request).
- Ran `/simplify` code review: removed unused `page` return from `harvest_set()`, moved `os.makedirs()` out of per-file hot path, eliminated duplicate gap-set computation in analyze script.

## Key decisions
- **Harvest ETDs as qdc, everything else as oai_dc** — **why:** qdc has richer thesis metadata (`thesis.degree.publication.type`, `thesis.degree.discipline`, `contributor.advisor`, `contributor.committeeMember`) that oai_dc lacks entirely. But qdc is missing `dc:language`, `dc:publisher`, `dc:format`, `dc:date`, `dc:source` on every record, and `dc:type` on ETD records. For non-ETD content, oai_dc wins.
- **Skip splitting constants into a shared `config.py`** — **why:** only two scripts in the project; `OAI_NS` and `DATA_DIR` duplication is trivial and a third file would add more friction than it saves.
- **Keep `ET.indent()` on output XML files** — **why:** makes the files human-readable for debugging, which was useful during analysis.

## Gotchas & things to remember
- **qdc field names have dots in the tag name** — e.g., `<dc:thesis.degree.publication.type>`, not a nested element. Look up by the literal tag string under the `dc:` namespace.
- **oai_dc `dc:type` uses canonical codes** (`article`, `doctoral_dissertation`, `thesis`) while **qdc `dc:type` uses human-readable labels** (`Article`, `Working Paper`, `Meeting Minutes`). Same records, different vocabularies.
- **Non-GC ETDs have no `dc:type` in qdc at all** — they only have `thesis.degree.level` + `thesis.degree.name`. But oai_dc for those same records has `dc:type=thesis`.
- **Cross-institution sets are incomplete** — `publication:etds`, `publication:pubs`, `publication:oers`, `publication:arch` collectively missing 1,701 records that exist in their institutional sub-sets. Biggest gap is `publication:arch` (1,074 missing). May be a Digital Commons indexing delay or classification difference.
- **GSA M&IE split, 12pt Arial** — unrelated, but this project uses 12pt Arial per user preferences; script already set up that way.
- **Primo VE discovery import profiles only allow one set or all sets** — blocking issue for the per-format harvest plan. Draft case already written for Ex Libris Support.
